    為什麼會有這篇文章?

    之前我發佈了一篇Hexo主題設定，因為內容只寫到如何換主題，但不同的主題有不同的類別或標籤設定，故另外文章做說明。

    這篇文章做什麼用的?

    這篇文章是來說明如何建立類別和標籤，方便尋找自己的文章。

    前置步驟&注意事項

    1.在開始實作這篇文章的內容前，請先實作完這篇文章如何安裝hexo+在github上建立個人網頁，因為這篇屬於上述文章的延伸。
    2.本文章是以Hexo主題設定裡選擇的主題做設定，若其他主題照本篇文章做有錯誤，請另外爬文，應該都大同小異，只是小細節不同罷了。
    3.瘟腥提醒！操作前請先備份一下自己的資料，不要佈一佈文章也跟著不見，最好是連原主題的設定以及hexo的設定檔都先備份一下。
    4.懂看config檔裡面的內容，這時候會英文就吃香了不少了呢！
    5.需要用點markdown語法

    從設定檔開啟類別menu

    當你在預覽主題時，有些可以看到Home、Archive、Tags、Categories、About等之類的頁面可顯示，但實際載下來後，就發現奇怪怎麼只有Home和Archive這兩個頁面的情況。
    其實原因在主題的設定檔，我們需要將頁面設定叫出來
    這是我的設定檔路徑:C:\Users\danie\hexotest\themes\hexo-theme-icalm

    開頭的地方有menu的設定，我們只需要將註解消除即可
    消除後就可以儲存退出了，我們就完成第一步驟了。

    建立類別或標籤

    當第一步驟完成時，你可以選擇先上傳至github上看結果，理論上會出現我們要的這幾個選項供我們切換。
    但是切換後的畫面是空白的或是按下去沒有反應，原因在沒有文章被分類，所以第二步驟我們就是要來操作如何歸納文章。

    首先，我們先打開CMD，並移動至hexo的資料夾路徑下

    然後就可以下指令來建立你要的分類選項
    因為標籤和類別我之前已經建立過了，所以我今天要建立About

    指令中的about，便是你要打的名稱。
    另外，在按enter前，先確認你主題設定檔的menu設定是不是跟你命名的相同，不同的話看是要自己將命令中的名稱改成跟設定檔一模一樣還是將設定檔裡的名稱改成跟命令的一樣。

    確認完畢後就可以按下enter，hexo就會幫你把標籤建立起來

    建立起來的檔案在這裡:C:\Users\danie\hexotest\source

    如果有正確地建立起資料夾，恭喜你第二步驟就結束了

    前置步驟
